Cel i motywacja

Cel pracy: Zaprojektowanie i implementacja systemu webowego wspomagającego zarządzanie działalnością gastronomiczną.

Problem: Tradycyjne metody zarządzania restauracją oparte na ręcznej dokumentacji są niewystarczające w dynamicznym środowisku gastronomicznym.

Rozwiązanie: GastroM — aplikacja webowa typu ERP, centralizująca dane i automatyzująca procesy zarządzania restauracją.

Panel główny (Dashboard)

Konfigurowalne centrum dowodzenia z 19 kafelkami statystyk:

  • Całkowita sprzedaż, zamówienia, zysk netto
  • Średnia wartość zamówienia i marża
  • Podział przychodów: gotówka vs karta
  • Alerty o niskim stanie magazynowym
  • Wykresy: top produkty, metody płatności
  • Ostatnie transakcje w czasie rzeczywistym

 Przedziały czasowe

Użytkownik wybiera zakres danych: dzienny, tygodniowy, miesięczny, roczny lub od początku działalności

 Drag & Drop

Personalizacja układu — przeciąganie, dodawanie i usuwanie kafelków

 Dane na żywo

Automatyczne odświeżanie i buforowanie danych z serwera

Zarządzanie magazynem

 Składniki

  • Dodawanie, edycja i usuwanie składników
  • Parametry: nazwa, jednostka (g, ml, szt.), cena jednostkowa
  • Uzupełnianie zapasów z automatycznym naliczaniem kosztu
  • Alerty o niskim stanie — próg ostrzeżenia dla każdego składnika

 Produkty sprzedażowe

  • Tworzenie produktów z recepturą (lista składników + ilości)
  • Cena sprzedaży i oznaczenie na wynos / w lokalu
  • Automatyczny koszt składników na podstawie receptury
  • Rentowność — analiza marży per produkt

Koszty stałe i raporty sprzedaży

 Koszty stałe

Ewidencja wydatków operacyjnych niezależnych od sprzedaży:

  • Czynsz, media, ubezpieczenia
  • Wynagrodzenia pracowników
  • Koszty uzupełniania magazynu
  • Typ kosztu, kwota, opis, data

Dane uwzględniane w kalkulacji zysku na dashboardzie

 Raporty sprzedaży

Kompleksowe raporty finansowe za wybrany okres:

  • Wybór przedziału dat (od — do)
  • Szczegółowe statystyki sprzedaży
  • Generowanie i archiwizacja na serwerze
  • Pobieranie w formacie PDF

Realizacja techniczna

Technologie, architektura i testowanie

Stos technologiczny

Frontend

  • React 18 + TypeScript
  • Material-UI v6
  • TanStack Query v5
  • React Router v7
  • Axios + Zod
  • Recharts (wykresy)
  • @dnd-kit (drag & drop)
  • Vite 6 (bundler)

Backend

  • Java + Spring Boot
  • Spring Security + JWT
  • Spring Data JPA
  • Hibernate ORM
  • REST API
  • Gradle (build)
  • Springdoc OpenAPI

Infrastruktura

  • PostgreSQL (baza danych)
  • Neon.com (hosting DB)
  • Render.com (deployment)
  • Git + GitHub
  • CI/CD (GitHub Actions)

Architektura: klient–serwer, komunikacja REST/JSON przez HTTP
