Cel i motywacja

Cel pracy: Zaprojektowanie i implementacja systemu webowego (aplikacji internetowej) do wspomagania działalności gastronomicznej.

Problem: Tradycyjne metody zarządzania działalnością gastronomiczną oparte na ręcznej dokumentacji są niewystarczające w dynamicznym środowisku gastronomicznym.

Rozwiązanie: GastroM - aplikacja webowa typu ERP, centralizująca dane i automatyzująca procesy zarządzania działalnością gastronomiczną. System dedykowany małym działalnościom gastronomicznym, dla których profesjonalne systemy ERP są zbyt kosztowne i złożone.

Realizacja techniczna

Technologie, architektura i testowanie

Model bazy danych

Relacyjna baza danych PostgreSQL z 9 tabelami:

  • employee - dane pracowników
  • transaction - transakcje sprzedażowe
  • product - produkty w ofercie
  • ingredient - składniki magazynowe
  • product_component - receptury
  • transaction_product - pozycje transakcji
  • report - raporty PDF
  • fixed_cost - koszty stałe
  • manager - konta użytkowników

Stos technologiczny

Frontend

  • React 18 + TypeScript
  • Material-UI v6
  • TanStack Query v5
  • React Router v7
  • Axios + Zod
  • Recharts (wykresy)
  • @dnd-kit (drag & drop)
  • Vite 6 (bundler)

Backend

  • Java + Spring Boot
  • Spring Security + JWT
  • Spring Data JPA
  • Hibernate ORM
  • REST API
  • Gradle (build)
  • Springdoc OpenAPI

Infrastruktura

  • PostgreSQL (baza danych)
  • Neon.com (hosting DB)
  • Render.com (deployment)
  • Git + GitHub
  • CI/CD (GitHub Actions)

Architektura: klient–serwer, komunikacja REST/JSON przez HTTP
